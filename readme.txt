项目简介:
设计并实现一个利用 Apache Kafka和 Temporal产生和消费消息的服务。该服务将包括两个
Temporal任务:一个是每分钟触发一次的消息产生任务，
另一个是消息消费任务，这是一个长期运行的工作流。
要求:
1.部署kafka和临时集群:在本地计算机中安装kafuka与临时集群
2.实现您的服务以运行两个cron工作流:
Cron Workflow 1:  每隔一分钟触发一次，启动 200 个并行运行的子工作流向 Kafka 主题发送消息。在启动新工作流之前终止任何先前存在的工作流。
Long Running Workflow 2:此工作流设置 10 个消费者不间断地从 Kafka 主题中。
连续消费消息。在启动新工作流之前终止任何先前运行的工作流。
3.运行您的服务:在本地计算机中 Temporalworker和cron工作流的设置和初始化。

介绍：
main.go 是生产者主函数

consumer.go 是消费者主函数
